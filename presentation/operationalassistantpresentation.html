The Problem

Operations questions don't live in one place

A grid operator asks "why are emergency vendor costs rising in our western region?" — the answer lives across three systems:

📊

Structured operations data

SCADA telemetry, work orders, outage history, financial ledger — millions of rows in Unity Catalog tables.

📄

Unstructured documents

Root cause analyses, corrective actions, vendor contracts, OEM manuals, escalation memos — PDFs scattered across SharePoint.

🧠

Tribal knowledge

"Talk to Mike, he handles those substations" — answers locked in heads, Slack threads, and meeting notes.

The fix: a supervisor agent that knows when to query data and when to read the document.

The Use Case

Cascadia Renewable Operations

A fictional 8-region renewable energy utility (wind / solar / battery) — facing rising emergency vendor costs and recurring asset failures concentrated in their Northwest footprint.

Product Walkthrough

Databricks Genie — natural language to SQL

Genie Spaces give business users a chat interface over curated data. Each space is scoped to one domain with a metric view as the semantic backbone.

⚡

Grid Operations Intelligence

15 measures · 7 sources · MTBF/MTTR, capacity factor, anomaly counts, grid instability risk

💰

Financial Performance

9 measures · revenue, OpEx, profit margin, market exposure, outage cost impact

🔧

Maintenance & Workforce

15 measures · preventive-vs-reactive ratio, technician utilization, parts cost

Powered by Unity Catalog Metric Views — declarative YAML for dimensions + measures, joins precomputed, MEASURE() function applies aggregation rules consistently.

SELECT region_name, MEASURE(forced_outage_count), MEASURE(revenue_loss_from_outages_usd)
FROM financial_performance_metrics GROUP BY region_name ORDER BY 3 DESC;
Product Walkthrough

Agent Bricks — multi-agent orchestration

Agent Bricks is the no-code platform for building production agents. We stitched together two Agent Bricks products:

📄

Knowledge Assistant

Instructed RAG over the 50-PDF UC Volume. Returns answers with inline citations and document references.

  • Auto-syncs new files
  • Up to 10 knowledge sources
  • Built-in feedback loop
🧠

Multi-Agent Supervisor

A Claude-Haiku-powered router that decides which sub-agent(s) to call, including in parallel, then synthesizes the final answer.

  • Up to 30 sub-agents / tools
  • Parallel tool execution
  • Streaming response API

Both deploy as standard model serving endpoints — same auth, scaling, monitoring as any other Databricks model.

Product Walkthrough

Databricks Apps — the frontend

FastAPI + static HTML, ~600 lines of Python, deployed in 30 seconds with the Databricks CLI. Auto OAuth via the workspace.

⚡ Live streaming

Server-Sent Events from the supervisor. Tool calls appear in the trace panel as they fire; the answer types in word-by-word.

🗂️ In-app Q&A cache

LRU keyed on (history + question) hash, 1-hour TTL. Repeat questions return in <1s with a "⚡ Cached" badge.

🔍 Routing inspector

Right-side panel shows every step the supervisor took: which Genie space, which document, what the LLM was reasoning about.

Genie Best Practices

From docs.databricks.com/genie/best-practices — 16 practices, distilled into the highest-impact themes for new Genie space authors.

📋 Define the space

Audience, purpose, topic boundary

🎯 Curate the data

Quality descriptions, ≤5 tables to start, ≤30 limit

🧪 Test & iterate

Benchmarks, user testing, refine instructions

Genie Best Practices · Data

Curate the data first, instructions second

1

Concise, well-documented datasets

Quality table + column descriptions in Unity Catalog are critical for Genie accuracy. The first lever — not the last.

2

Start small

Begin with ≤5 tables. Iterate based on real user feedback rather than what you think they'll ask.

3

Work within the 30-table limit

If you need more, pre-join into views. A smaller, focused space outperforms a bloated one.

4

Add metadata + synonyms

Synonyms scoped to your space help Genie match user vocabulary to your column names.

5

Domain expert defines the space

Data analysts proficient in SQL should curate — they understand both data and the insights worth surfacing.

6

Define the space's purpose

Audience + topic boundary. "Marketing analytics" beats "all sales data" for accuracy.

7

Use prompt matching

Auto-corrects spelling and matches values to user phrasing without you writing rules.

8

One topic per space

If you're tempted to add unrelated tables, build a second space and let users pick.

Genie Best Practices · Instructions

Prefer SQL examples over text instructions

9

SQL expressions > example SQL > text instructions

Hierarchy: SQL expressions for metrics, example SQL for complex questions, text instructions only as a last resort.

10

Focused examples beat lengthy guidance

5 great example SQLs outperform 5 paragraphs of text. Each example teaches a pattern.

11

Write clear, specific text instructions

Replace vague guidance with concrete scenarios and expected behavior. No "be helpful" — specify when and how.

12

Avoid conflicting instructions

SQL examples and text guidance must align on details like rounding, time zones, default filters.

13

Prompt Genie to ask clarifying questions

Tell it explicitly when it should pause and ask, and what specific questions to ask.

14

Customize summaries

Genie's natural-language summary alongside results can be tuned with dedicated instructions.

15

Test & adjust

Inspect generated SQL. Add benchmark questions. Score systematically. Don't ship blind.

16

Real user testing

Recruit business users with clear expectations. Their unscripted questions reveal the gaps.

Best Practices · 2 of 3

Knowledge Assistant Best Practices

"Instructed retriever" approach that goes beyond traditional RAG. The key: feedback loops and curated knowledge sources.

📚 ≤10 sources per assistant

Use incremental sync, not full re-ingest

👥 Collaborative feedback

SMEs label examples, you tune via natural-language guidelines

🔐 Permission scoping

CAN_MANAGE for editors, CAN_QUERY for end users

Knowledge Assistant · Configuration

Get the file plumbing right

Supported sources

  • UC Volume: .txt, .pdf, .md, .ppt/.pptx, .doc/.docx
  • UC Table: requires content (BINARY/STRING) + metadata struct
  • 50 MB cap per file (auto-excluded if larger)
  • Vector Search: only databricks-gte-large-en embedding model is supported

Iterative improvement

A

Gather labeled examples

SMEs mark good vs bad answers. Use the agent configuration page.

B

Natural-language guidelines

Tune behavior without writing code — describe what you want in plain English.

C

Incremental sync

When source files change, sync only the deltas — not the whole corpus.

D

Disable AI Guardrails

On the embedding endpoint specifically — required for proper KA function.

Best Practices · 3 of 3

Multi-Agent Supervisor Best Practices

The supervisor coordinates Genie spaces, agent endpoints, and Unity Catalog functions. The quality of its routing depends almost entirely on how well you describe each tool.

📝 Detailed tool descriptions

"As much detail as possible" — the supervisor routes on these

🔄 Iterative refinement

Test → expert feedback → natural-language guidelines

🛡️ Permission alignment

End users need explicit access to each sub-agent

Multi-Agent Supervisor · Configuration

Start with the essentials, then iterate

Initialize correctly

1

Up to 30 integrated agents/tools

Plenty for most enterprise use cases. Quality of descriptions matters more than count.

2

Detailed sub-agent descriptions

Each description is what the supervisor sees when deciding to route. Vague descriptions = wrong tool calls.

3

Clear top-level instructions

"When user asks about X, prefer tool Y. When they need numbers AND documents, call both in parallel."

Test like a power user

4

Use the AI Playground

Enter complex multi-part questions. Watch task delegation. Enable AI Judge + synthetic task generation.

5

Add response guidelines when prompted

The platform asks for them — they're how you teach the supervisor non-obvious patterns.

6

Workspace requirements

Serverless compute, Unity Catalog, Model Serving, supported region. databricks-gte-large-en endpoint with AI Guardrails + rate limits disabled.

Performance Tuning

What actually moves the needle

Based on real benchmarking of this exact stack: simple questions go from 25s → ~10s, cross-source from 100s → ~30-40s.

Change Where Impact
Switch supervisor LLM to Haiku 4.5 Agent Bricks → Multi-Agent System → Model 40-60% faster
Enable parallel tool calls Agent Bricks → Advanced 20-40% on multi-tool
Trim system prompt to <500 tokens Agent Bricks → Instructions 5-15% per turn
KA top_k = 5-8 (not default 20) Knowledge Assistant settings 10-25% on doc Q&A
